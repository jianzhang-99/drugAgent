# 前端待开发清单

## 1. 文档说明

本文档用于梳理当前 `drug-agent-web` 前端项目尚未完成的能力，并给出建议的开发优先级，便于后续分工、排期与联调。

当前前端已经完成了基础框架搭建，包含以下页面和公共布局：

- 工作台 `Agent Hub`
- 任务看板
- 合规知识库
- 顶部导航与侧边栏布局

现阶段整体状态为“页面框架已搭建，业务闭环尚未完成，部分能力仍为静态演示数据”。

---

## 2. 当前已具备的前端基础

### 2.1 页面框架

- 已完成基础路由划分
- 已完成工作台、任务看板、知识库三个主页面壳子
- 已完成公共布局、导航栏、侧边栏、卡片风格与基础交互样式

### 2.2 已接入的基础能力

- 已配置 Vue 3 + Vite + Vue Router
- 已接入 Element Plus
- 已封装 Axios 请求实例
- 已封装部分 Agent 和标书审查接口
- 已预留 SSE 流式对话调用能力

### 2.3 当前存在的总体问题

- 页面中的大量内容仍为静态 mock 数据
- 关键业务流程只有入口，没有完整的结果承接页面
- 前后端参数对接存在不一致问题
- 缺少任务详情、解析详情、知识库管理等核心业务页面
- 缺少错误处理、空状态、权限、状态管理等工程化能力

---

## 3. P0 必须优先完成

P0 目标：先把“能用”做出来，形成最小可用闭环。

### 3.1 工作台文本对话打通

**目标**

让用户在工作台输入问题后，能够成功调用后端 Agent 对话接口，并看到返回结果。

**当前问题**

- 前端提交文本时传递字段为 `message`
- 后端接口要求字段为 `query`
- 当前纯文本对话链路实际不可用

**开发内容**

- 修正前端请求参数字段
- 增加对话提交校验
- 增加发送中状态与失败提示
- 返回结果在页面中展示

**交付结果**

- 用户可直接在工作台完成一次非流式问答

### 3.2 聊天记录展示区

**目标**

工作台不仅能发请求，还要能展示上下文消息记录。

**开发内容**

- 接入消息列表区域
- 接入用户消息气泡
- 接入 AI 回复气泡
- 接入加载中状态
- 支持基础 Markdown 渲染

**交付结果**

- 工作台具备基础聊天窗口能力

### 3.3 SSE 流式输出接入

**目标**

让 AI 回复支持流式展示，符合 Agent 产品体验。

**开发内容**

- 接入现有 SSE API
- 支持逐段拼接展示回复内容
- 支持流式结束状态
- 支持异常中断提示
- 支持重复提问后的消息追加

**交付结果**

- 用户可看到流式回复过程

### 3.4 标书上传校验

**目标**

保证上传入口符合后端约束，减少无效请求。

**开发内容**

- 限制至少上传 2 份文件
- 限制仅允许 `.docx`
- 增加文件大小限制
- 增加重复文件提示
- 增加非法文件拦截提示

**交付结果**

- 上传入口可控，用户操作错误能被前端提前拦截

### 3.5 标书上传创建任务闭环

**目标**

上传文件后不仅创建任务，还要能在前端留存任务上下文。

**开发内容**

- 上传成功后保存 `caseId`
- 保存文档 `docId` 列表
- 跳转看板时带上任务标识
- 能够识别“刚创建的任务”

**交付结果**

- 上传动作与任务看板形成基本闭环

### 3.6 任务看板改造为真实数据

**目标**

任务看板不再使用纯静态数据，而是展示真实任务或至少展示当前会话创建出的任务。

**开发内容**

- 去除演示写死数据
- 接入后端任务列表接口
- 如果后端暂未提供列表接口，可先以前端状态承接已创建任务
- 展示任务名称、状态、创建时间、场景类型

**交付结果**

- 任务看板具备基础可用性

## 4. P1 核心功能开发

P1 目标：把“能用”升级为“能完成真实业务”。

### 4.1 标书解析流程接入

**目标**

创建任务后，前端可以触发并展示文档解析进度和结果。

**开发内容**

- 调用文档解析接口
- 展示每个文档的解析状态
- 展示解析成功与失败信息
- 展示解析结果的基础摘要

**交付结果**

- 用户可看到上传文档是否真正进入解析流程

### 4.2 新增任务详情页

**目标**

为每个任务提供独立详情页面，承接后续结果展示。

**建议展示内容**

- 任务基础信息
- 上传文件列表
- 文档解析进度
- 风险识别结论
- 命中规则摘要
- 证据链展示
- 处理时间与执行日志

**交付结果**

- 用户可从看板进入单任务详情查看完整信息

### 4.3 任务状态实时刷新

**目标**

任务从创建到完成的状态变化可实时感知。

**开发内容**

- 支持轮询或 SSE 刷新
- 支持处理中、已完成、失败、待复核等状态展示
- 支持进度条或步骤条展示

**交付结果**

- 看板和任务详情可反映真实执行状态

### 4.4 看板筛选与搜索

**目标**

让任务看板具备真实管理能力，而不仅是展示页面。

**开发内容**

- 按场景筛选
- 按任务状态筛选
- 按风险等级筛选
- 按时间范围筛选
- 关键字搜索任务

**交付结果**

- 看板可用于任务定位与管理

### 4.5 看板操作按钮落地

**目标**

让任务卡片上的操作按钮具备实际意义。

**开发内容**

- `查看报告` 跳转到任务详情或报告页
- `立即复核` 跳转人工复核视图
- `实时日志` 展示任务执行日志
- `查看细节` 展示命中证据与任务摘要

**交付结果**

- 看板中的按钮形成完整操作链路

### 4.6 工作流卡片点击行为

**目标**

首页常用工作流不再只是装饰卡片。

**开发内容**

- 点击工作流卡片后预填提示词
- 或跳转对应场景页面
- 或直接打开上传/分析入口

**交付结果**

- 首页工作流卡片可真正引导用户进入场景操作

### 4.7 上传后的任务定位能力

**目标**

用户上传后跳转到看板时，能快速找到刚才创建的任务。

**开发内容**

- 携带任务 ID 跳转
- 默认高亮当前任务
- 支持自动滚动定位

**交付结果**

- 用户体验更连贯，避免“创建成功但找不到任务”

## 7. 推荐开发顺序

建议按以下顺序推进：

1. 工作台文本对话打通
2. 聊天记录区与流式输出接入
3. 标书上传校验与任务创建闭环
4. 任务看板真实数据展示
5. 任务详情页与解析结果展示
6. 看板筛选、实时刷新、任务定位
7. 知识库导入、列表、删除、搜索
8. 规则组和字典库管理
9. 审计日志、偏好配置、帮助页
10. 权限、状态管理、工程化优化

## 8. MVP 最小可用版本建议

如果当前目标是尽快形成一个可演示、可联调的最小可用版本，建议优先只做以下 4 项：

### 8.1 工作台聊天可用

- 用户输入问题
- Agent 返回结果
- 页面可展示消息记录

### 8.2 标书上传可用

- 支持上传多个标书文件
- 支持基础文件校验
- 能创建审查任务

### 8.3 任务看板可看真实任务

- 能展示新建任务
- 能看到任务状态
- 能进入详情

### 8.4 任务详情可看解析和风险结果

- 能看到文档解析状态
- 能看到风险判断结果
- 能看到基础证据内容

---

## 9. 建议的前端任务拆分

为了方便分工，建议将前端拆成以下几个模块并行开发：

### 模块 A：工作台与对话

- 工作台输入交互
- 聊天记录区
- SSE 流式输出
- 工作流卡片行为

### 模块 B：任务链路

- 标书上传
- 任务创建
- 任务看板
- 任务详情
- 状态刷新

### 模块 C：知识库管理

- 知识文档列表
- 文档导入
- 文档删除与重建索引
- 规则组管理
- 实体字典管理

### 模块 D：系统支撑能力

- 审计日志
- 配置与帮助页
- 状态管理
- 权限控制
- 工程化优化

---

## 10. 结论

当前前端项目已经完成了较好的视觉框架和页面骨架，但距离“可支撑真实监管业务”的版本还有较多业务功能待补齐。

建议采用“先闭环、后完善”的策略：

- 第一阶段先打通工作台、上传、看板、详情 4 个核心链路
- 第二阶段补齐知识库管理与筛选操作
- 第三阶段完善审计、配置、权限和工程化能力

这样可以最快形成一个可联调、可演示、可继续扩展的前端版本。
