一句话流程：用户通过前端输入需求 → 后端预处理并由大模型识别意图 → Spring AI编排调用Skill/MCP/RAG等工具 → 生成结构化或自然语言结果 → 前端呈现订单卡片/地图、航线/对话内容

`AI 小吨 智能体系统`拟采用 **LangChain 为核心编排框架**，结合 **RAG 向量知识库 + Tool 执行能力 + 多模型推理**，构建具备 **理解、规划、执行、记忆** 能力的行业级智能体。

## 应用 / 接入层

**作用**：

和人、系统交互

*   **多渠道接入**：支持用户从不同终端访问
    
*   **统一入口**：所有请求通过标准化接口进入系统
    
*   **协议转换**：处理HTTP/WebSocket/SSE等协议
    
*   **会话管理**：维护对话上下文和状态
    
*   **流式响应**：支持实时打字机效果

### 4.1 Agent Skill

> 定位：领域专家模块，封装完整的任务执行能力

**核心特性**

*   每个 Skill 是一个独立的能力单元，包含完整的任务流程和最佳实践
    
*   预置专业知识和执行逻辑，如代码生成、设计创作、数据分析等
    
*   支持组合调用，实现复杂任务的协同完成
    

**应用场景**

*   业务信息录入（货源发布、订单创建、航次管理）【专业客服】
    

*   GitLab 代码评审（代码质量检查、代码统计分析）【专业运维】
    

*   业务指标统计（运输效率、订单完成率、成本分析）【专业分析】
    

*   地图查船找船（船舶位置追踪、航线规划、港口查询）【专业调度】
    

### 4.2 MCP（Model Context Protocol）

> 定位：标准化的工具访问协议，统一外部资源调用，相当于AI中的“USB/API”

**核心能力**

*   提供统一接口规范，连接 Agent 与外部工具/服务
    
*   支持多种工具类型：数据库、API、文件系统、第三方服务
    
*   实现工具的即插即用，降低集成复杂度
    

**典型应用**

*   数据库操作（MySQL-MCP）
    
*   文件系统访问（本地文件操作）
    
*   第三方服务集成（高德地图、天气、Jira、Gitlab）
    

> **与 Skill 的区别和联系**

> Skill 专注于"做什么"（任务逻辑），MCP 专注于"用什么"（工具调用） 两者配合实现完整的任务执行链路

### 4.3 Function Call

> 定位：结构化的函数调用机制，实现精确的能力扩展

**核心特性**

*   基于 JSON Schema 定义函数签名和参数
    
*   支持同步/异步调用，返回结构化数据
    
*   可与 Skill 和 MCP 协同工作
    

### 4.4 本地知识库（RAG）

> 定位：行业智能的核心，提供领域知识检索与增强。核心：**RAG + 业务感知**

**架构流程**

```plaintext
文档输入 → 文本切分 → 向量化 → 向量库存储 → 语义检索 → 相关性重排 → 注入 Prompt
```

**业务流程**

```plaintext
┌────────────┐
│  业务系统   │  运单 / 船舶 / 合同 / 对话 / 消息
└─────┬──────┘
      │
      ▼
┌────────────┐
│ 知识接入层  │ 文档解析 / OCR / ASR / 结构化抽取
└─────┬──────┘
      │
      ▼
┌────────────┐
│ 向量生成层  │ Embedding（通义 / Qwen / OpenAI / 本地）
└─────┬──────┘
      │
      ▼
┌────────────────────┐
│ Milvus 向量知识库   │ 语义召回（主）
└─────┬──────────────┘
      │
      ▼
┌────────────┐
│  重排 & 过滤 │ 权限 / 时间 / 角色 / 置信度
└─────┬──────┘
      │
      ▼
┌────────────┐
│   LLM 推理  │ AI 小吨
└────────────┘
```

**核心组成**

1.  **知识存储**
    

*   向量化存储：支持语义级检索，而非关键词匹配
    
*   多源知识整合：文档、政策、规则、FAQ 等一体化管理
    
*   动态更新机制：支持增量添加和实时更新
    

1.  **技术选型**
    
    | 环节 | 推荐方案 | 说明 |
    | --- | --- | --- |
    | 文档解析 | Unstructured / PyPDF2 / python-docx | 支持 PDF、Word、Excel 等多格式 |
    | Embedding 模型 | bge-large-zh（中文）<br>bge-m3（多语言） | 高质量向量表示，适配中文场景 |
    | 向量数据库 | Milvus | 高性能向量检索，支持分布式部署 |
    | RAG 框架 | LangChain | 提供完整的 Retriever、Reranker 组件 |
    
2.  **优化策略**
    
    *   文本切分：采用 Semantic Chunking，保持语义完整性（推荐 500-1000 tokens/chunk）
        
    *   混合检索：向量检索 + 关键词检索（BM25），提升召回率
        
    *   重排序：使用 Reranker 模型（如 bge-reranker-large）精排 Top-K 结果
        
    *   提示工程：动态构建 Few-shot 示例和上下文窗口
        
3.  **应用场景**
    

*   **平台操作知识库**：货主端操作手册、船主端使用指南、运吨吨产品功能介绍
    

*   **行业法规合规**：水路运输法规、港口管理条例、货运安全规范、海事监管要求
    

*   **业务流程指导**：货源发布流程、订单操作规范、航次管理指引、结算流程说明
