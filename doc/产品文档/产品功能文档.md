# Drug-Agent 药品智能监管系统 — 产品功能文档

> **版本**：v1.0 | **日期**：2026-02-28

---

## 一、产品定位

一句话描述：**基于大模型的药品使用智能监管与合规审查系统**

| 目标用户 | 使用场景 |
|---------|---------|
| 医院药品管理科 | 分析药品用量趋势、识别异常用药 |
| 药品监管人员 | 审查药品相关文件是否合规 |
| 医疗机构管理层 | 查看药品监管分析报告，辅助决策 |

---

## 二、功能清单

### 模块A：药品数据监管

| 编号 | 功能点 | 优先级 | 说明 |
|:----:|--------|:-----:|------|
| A1 | Excel 批量导入 | P0 | 上传 Excel，批量录入药品使用记录到数据库 |
| A2 | 数据查询与筛选 | P0 | 按药品名/日期/科室/分类等条件分页查询 |
| A3 | AI 智能分析 | P0 | 选择药品和时间段，AI 自动分析趋势、异常、风险 |
| A4 | 分析报告展示 | P1 | 结构化展示分析结果（趋势图 + 异常列表 + 建议） |
| A5 | Excel 模板下载 | P2 | 提供标准导入模板下载 |

### 模块B：文件合规对话

| 编号 | 功能点 | 优先级 | 说明 |
|:----:|--------|:-----:|------|
| B1 | 文件上传 | P0 | 支持上传 PDF / Excel / Word 格式的药品文件 |
| B2 | 文件自动解析 | P0 | 系统自动提取文件文本内容 |
| B3 | 合规对话 | P0 | 基于文件内容与 AI 多轮对话，判断合规性 |
| B4 | 对话历史查看 | P1 | 查看历史会话列表及消息记录 |
| B5 | 文件列表管理 | P1 | 查看已上传文件，支持删除 |

### 模块C：法规知识库（RAG）

| 编号 | 功能点 | 优先级 | 说明 |
|:----:|--------|:-----:|------|
| C1 | 法规文件上传 | P0 | 上传法规文件，自动切片+向量化入库 |
| C2 | 对话自动检索 | P0 | 合规对话时自动检索相关法规条款注入上下文 |
| C3 | 知识库列表 | P1 | 查看已入库的法规文件及状态 |
| C4 | 知识库删除 | P2 | 删除过期或错误的法规文件 |
| C5 | 检索效果测试 | P2 | 输入关键词，查看检索到的法规片段 |

### 模块D：通用能力

| 编号 | 功能点 | 优先级 | 说明 |
|:----:|--------|:-----:|------|
| D1 | 多轮对话 | P0 | 支持上下文连贯的连续对话 |
| D2 | 会话管理 | P1 | 新建/切换/删除会话 |
| D3 | 流式输出 | P2 | SSE 逐字输出，提升体验 |

### 模块E：前端页面

| 编号 | 功能点 | 优先级 | 说明 |
|:----:|--------|:-----:|------|
| E1 | 药品监管面板 | P0 | 数据导入 + 查询 + AI分析的操作界面 |
| E2 | 合规对话界面 | P0 | 类 ChatGPT 的对话 UI，支持文件上传 |
| E3 | 知识库管理页 | P1 | 法规文件上传/列表/删除界面 |

## 五、技术实现里程碑

### 里程碑总览

```
M1         M2         M3         M4         M5         M6
 │──────────│──────────│──────────│──────────│──────────│
 基础搭建    数据监管    文件合规    RAG知识库   前端集成    测试上线
 (3天)      (5天)      (5天)      (4天)      (5天)      (3天)
```

### 里程碑节点详表

#### M1 — 基础搭建（预计3天）

| 任务 | 涉及功能 | 产出物 | 验收标准 |
|------|---------|--------|---------|
| 数据库建表 | 全局 | SQL 脚本 + flyway 迁移 | 5张表建好，可正常读写 |
| 统一响应封装 | 全局 | `Result<T>` 类 | 所有接口返回统一JSON格式 |
| 文件上传通用模块 | B1/C1 | `FileService` | 可上传文件到 `uploads/` 目录 |
| 千问调用封装完善 | 全局 | `QwenService` 增强 | 支持多轮消息、系统Prompt |

#### M2 — 药品数据监管模块（预计5天）

| 任务 | 涉及功能 | 产出物 | 验收标准 |
|------|---------|--------|---------|
| Excel 导入接口 | A1 | `DrugController.import` | 上传 Excel 成功入库，异常行有提示 |
| 数据查询接口 | A2 | `DrugController.list` | 支持多条件筛选+分页 |
| 数据统计聚合 | A3 | `DrugMonitorService` | 按药品+时间段输出统计摘要 |
| AI 分析 Prompt 编写 | A3 | Prompt 模板 | 模型返回结构化分析报告 |
| AI 分析接口联调 | A3/A4 | `DrugController.analyze` | 输入药品+时间段，返回完整分析报告 |

#### M3 — 文件合规对话模块（预计5天）

| 任务 | 涉及功能 | 产出物 | 验收标准 |
|------|---------|--------|---------|
| 文件解析引擎 | B2 | `FileParserService` | PDF/Excel/Word 均可提取文本 |
| 多轮对话管理 | D1 | `SessionManager` | 对话上下文连贯，历史消息正确维护 |
| 合规对话 Prompt | B3 | Prompt 模板 | 模型能基于文件内容回答合规问题 |
| 合规对话接口 | B3/B4 | `ComplianceController` | 上传文件→发问→得到合规判断 |
| 对话历史接口 | B4/D2 | `SessionController` | 可查看会话列表和历史消息 |

#### M4 — RAG 知识库模块（预计4天）

| 任务 | 涉及功能 | 产出物 | 验收标准 |
|------|---------|--------|---------|
| 文本分块(Chunking) | C1 | `TextSplitter` | 文本正确切片，有重叠 |
| Embedding 调用封装 | C1 | `EmbeddingService` | 文本→向量化成功 |
| 向量存储与检索 | C2 | `RAGEngine` | 输入问题，返回Top-K相关片段 |
| 知识库管理接口 | C1/C3/C4 | `KnowledgeController` | 上传/列表/删除法规文件 |
| 合规对话集成RAG | C2 | 修改 `ComplianceService` | 对话时自动检索法规并注入Prompt |

#### M5 — 前端集成（预计5天）

| 任务 | 涉及功能 | 产出物 | 验收标准 |
|------|---------|--------|---------|
| Vue 项目初始化 | E全局 | 前端框架搭建 | 可运行dev server |
| 药品监管面板页面 | E1 | 数据表格+导入+分析 | 完整走通导入→查询→分析流程 |
| 合规对话页面 | E2 | 对话UI+文件上传 | 完整走通上传→对话→查看历史 |
| 知识库管理页面 | E3 | 文件上传+列表 | 可上传法规文件、查看列表 |
| 联调与样式优化 | E全局 | 整体优化 | 页面美观、交互流畅 |

#### M6 — 测试与上线（预计3天）

| 任务 | 涉及功能 | 产出物 | 验收标准 |
|------|---------|--------|---------|
| 端到端测试 | 全局 | 测试报告 | 核心功能全部走通无报错 |
| 异常处理完善 | 全局 | 统一异常处理 | 各种异常有友好提示 |
| 文档完善 | 全局 | README + 部署文档 | 他人可按文档跑起来 |

## 六、Demo 交付标准

Demo 演示时需要跑通以下 **3 条核心链路**：

| # | 链路 | 演示步骤 |
|---|------|---------|
| 1 | 药品监管 | 上传Excel → 查看数据 → AI分析 → 看到报告和风险等级 |
| 2 | 合规对话 | 上传文件 → 提问合规问题 → AI引用法规回答 |
| 3 | 知识库 | 上传法规PDF → 入库成功 → 合规对话时能检索到 |
