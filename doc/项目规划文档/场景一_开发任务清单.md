标书智能识别 Demo 开发任务清单

一、项目目标

在最小版本里，完成一个可演示的流程：

上传/选择样本
→ 解析标书
→ 提取段落/表格/关键字段
→ 做相似度分析与风险判定
→ 展示证据
→ 生成报告


⸻

二、任务分阶段

建议拆成 5 个阶段：
1.	样本和规则准备
2.	后端核心能力
3.	前端演示页面
4.	LLM解释与报告
5.	联调与演示优化

⸻

三、详细任务清单

⸻

阶段 1：样本和规则准备

这个阶段最重要，因为 Demo 的说服力先来自样本，而不是代码。

1.1 样本整理

任务：
•	解压并整理三组样本
•	统一文件命名
•	给每个样本加标签

建议命名：
•	group1_compliance_a.docx
•	group1_compliance_b.docx
•	group2_source.docx
•	group2_suspect.docx
•	group3_bid_a.docx
•	group3_bid_b.docx

产出：
•	样本目录
•	样本说明文档

⸻

1.2 人工标注演示证据

任务：
•	从第二组中挑出 5–10 组“明显语义相似”的对照段落
•	从第三组中挑出 5–10 个“围标高风险特征”
•	从第一组中挑出 3–5 个“正常差异点”

产出：
•	demo_evidence.json

示例结构：

[
{
"scene": "plagiarism",
"doc_a": "group2_source.docx",
"doc_b": "group2_suspect.docx",
"type": "semantic_similarity",
"a_text": "...",
"b_text": "...",
"label": "high"
}
]


⸻

1.3 规则清单整理

任务：
整理一版最小规则表。

建议先只做这几类：

抄袭规则
•	段落 embedding 高相似
•	标题顺序一致
•	固定短语重复
•	错别字共现

围标规则
•	联系人相同
•	电话/邮箱相近
•	团队角色重复
•	报价差异极小
•	表格结构高度一致
•	目录结构高度一致

合规低风险规则
•	技术方案明显不同
•	团队不同
•	报价结构差异明显

产出：
•	rules.md
•	rules_config.json

⸻

阶段 2：后端核心能力

这是 Demo 的主干。

⸻

2.1 项目初始化

任务：
•	创建 FastAPI 项目
•	定义目录结构
•	增加基础配置

建议目录：

backend/
app/
api/
core/
services/
models/
utils/
data/

子任务：
•	建虚拟环境
•	安装依赖
•	配置 .env
•	增加日志

产出：
•	后端项目骨架

⸻

2.2 文件上传与样本读取

任务：
•	支持上传 docx
•	支持读取预置样本
•	保存文件元信息

接口建议：
•	POST /api/files/upload
•	GET /api/files/samples

产出：
•	上传接口
•	样本列表接口

⸻

2.3 docx 解析模块

任务：
•	解析段落
•	解析标题
•	解析表格
•	提取文档顺序结构

推荐：
•	python-docx

解析结果建议统一成：

{
"doc_id": "xxx",
"title": "xxx",
"paragraphs": [],
"tables": [],
"sections": [],
"metadata": {}
}

子任务：
•	提取普通段落
•	根据样式识别标题
•	提取表格行列
•	过滤空白段落
•	保留段落顺序索引

产出：
•	docx_parser.py

⸻

2.4 关键字段提取

任务：
从解析结果中抽取：
•	联系人
•	电话
•	邮箱
•	公司名称
•	项目经理
•	团队成员
•	金额
•	服务时效
•	风险控制关键词

实现方式：
•	正则 + 关键词规则
•	后续再加 LLM

产出：
•	field_extractor.py

⸻

2.5 文档分块模块

任务：
把文档拆成适合比对的单元：
•	段落块
•	章节块
•	表格块

建议先按：
•	段落级
•	标题下的章节级

子任务：
•	给每个 chunk 编号
•	记录所属章节
•	保留原始文本引用

产出：
•	chunker.py

⸻

2.6 词面相似度模块

任务：
实现基础文本相似度。

建议方法：
•	TF-IDF cosine
•	n-gram overlap
•	编辑距离

产出：
•	lexical_similarity.py

⸻

2.7 语义相似度模块

任务：
实现 embedding 相似度。

建议方案：
•	sentence-transformers
•	先离线计算段落向量

子任务：
•	加载 embedding 模型
•	生成 chunk embedding
•	计算相似度矩阵
•	返回 top-k 相似片段对

产出：
•	semantic_similarity.py

⸻

2.8 结构相似度模块

任务：
做最小结构比对。

建议先做：
•	标题顺序相似
•	章节命名相似
•	表格字段相似

子任务：
•	提取标题树
•	计算标题序列相似度
•	表格表头相似度

产出：
•	structure_similarity.py

⸻

2.9 围标规则引擎

任务：
把第三组样本中的高风险模式规则化。

先做最小规则：
•	联系人相同
•	电话相似
•	邮箱相似
•	团队成员重叠
•	报价差额异常
•	表格模板相似
•	稀有词/错字共现

输出：

{
"rule_hits": [
{
"rule": "shared_contact",
"score": 0.9,
"evidence": "联系人姓名相同"
}
]
}

产出：
•	risk_rules.py

⸻

2.10 综合判分模块

任务：
把各模块结果融合成总风险分。

建议最小公式：
•	词面相似 30%
•	语义相似 35%
•	结构相似 15%
•	围标规则 20%

输出：
•	总分
•	风险等级
•	命中原因列表

产出：
•	risk_scorer.py

⸻

2.11 分析总流程编排

任务：
串起完整分析流程：

读取文档
→ 解析
→ 提取字段
→ 分块
→ 相似度计算
→ 规则命中
→ 风险评分
→ 输出结果

接口建议：
•	POST /api/analyze/compare

产出：
•	analysis_service.py

⸻
