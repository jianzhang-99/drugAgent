标书智能识别 Demo 最小 PRD

1. 项目名称

医疗招投标标书智能识别 Demo

副标题可写：

面向围标、串标、抄袭风险的智能审查演示系统

⸻

2. 项目背景

在医疗监管与合规审查中，招投标环节存在较高的违规风险，例如：
•	围标
•	串标
•	标书抄袭
•	模板化洗稿
•	陪标

传统查重工具主要依赖关键词匹配或简单文本比对，难以应对以下问题：
•	标书篇幅长，通常几十到数百页
•	文档格式复杂，包含表格、章节、报价项等非结构化内容
•	存在同义替换、语序调整、模板改写等“洗稿”行为
•	合法法规引用、行业标准引用容易被误判为抄袭

因此，需要一个能同时识别 文本相似、结构相似、围标特征 的智能审查方案。

⸻

3. Demo 目标

本 Demo 不追求完整生产能力，只验证三件事：

目标 1：能识别“洗稿式抄袭”

即使文本经过改写，系统仍能识别语义相似。

目标 2：能识别“围标/陪标风险特征”

不仅比较文本，还能发现联系人、团队、报价、结构模板等异常关联。

目标 3：能输出“可解释证据”

不是只给一个相似度分数，而是输出：
•	哪些段落相似
•	哪些表格/字段相似
•	哪些围标特征命中
•	风险判断理由

6. 核心业务场景

场景 A：合规样本识别

输入两份正常独立编制标书，系统应输出：
•	相似度较低
•	无明显围标特征
•	风险等级低

场景 B：抄袭/洗稿识别

输入原始文本与改写后的疑似抄袭文本，系统应输出：
•	高语义相似
•	结构相似
•	命中疑似抄袭证据
•	风险等级中/高

场景 C：围标/陪标识别

输入恶意围标样本，系统应输出：
•	文本/结构/字段层面的同源性
•	报价异常或陪标特征
•	团队/联系人重复或相近
•	风险等级高

⸻

7. 产品核心价值

相较传统查重工具，本 Demo 的核心价值是：

1）不只看字面重复

还能识别：
•	近义替换
•	语序调整
•	洗稿改写
•	模板套写

2）不只做文本查重

还能识别：
•	表格结构相似
•	章节顺序相似
•	团队/联系人重叠
•	报价协同行为

3）不只给结论

还能输出：
•	证据片段
•	命中特征
•	风险原因
•	审查解释

⸻

8. 功能需求

8.1 文件上传

功能说明

用户上传 2 份或多份 .docx 标书样本。

输入
•	docx 文件
•	可选：选择演示场景

输出
•	文件解析状态
•	文件列表

⸻

8.2 文档解析

功能说明

将标书解析为可分析的数据单元。

解析内容
•	标题
•	正文段落
•	表格
•	联系方式
•	团队角色
•	报价字段
•	服务承诺字段
•	风险控制字段

输出结构

每份文档拆解为：
•	文档基础信息
•	段落列表
•	表格列表
•	关键字段列表

⸻

8.3 相似度分析

功能说明

对标书内容做多维分析。

分析维度

文本层
•	词面相似
•	n-gram 重合
•	编辑距离
•	高频短语复用

语义层
•	段落 embedding 相似度
•	改写语义相似识别

结构层
•	标题顺序相似
•	段落组织相似
•	表格字段相似

围标风险层
•	联系人/电话/邮箱相近
•	团队角色重复
•	报价梯度异常
•	罕见错误共现
•	模板化痕迹明显

⸻

8.4 风险判定

功能说明

根据相似度特征和规则特征，输出风险等级。

风险等级
•	低风险：独立编制可能性高
•	中风险：存在疑似抄袭或模板化改写
•	高风险：存在围标/陪标/同源编制明显迹象

判定方式
•	规则打分
•	相似度打分
•	LLM 解释辅助

⸻

8.5 证据展示

功能说明

展示命中的关键证据，便于投资人直观看到系统价值。

展示内容
•	相似段落对照
•	相似表格字段对照
•	命中的围标特征标签
•	对应页码/章节
•	风险说明

⸻

8.6 智能报告生成

功能说明

输出一份简要审查报告。

报告内容
•	文档比对对象
•	风险等级
•	命中特征摘要
•	相似证据列表
•	审查结论说明

⸻

10. 最小技术方案

10.1 技术架构

前期采用最轻量实现：
•	前端：React / Next.js
•	后端：Python FastAPI
•	文档解析：python-docx
•	相似度分析：
•	TF-IDF / n-gram
•	编辑距离
•	embedding 相似度
•	向量模型：sentence-transformers 或外部 embedding API
•	LLM：用于风险解释和报告生成

⸻

10.2 核心处理流程

上传标书
→ 解析 docx
→ 提取段落/表格/字段
→ 多维相似度分析
→ 命中规则特征
→ 输出风险等级
→ 生成证据视图与报告


⸻

•	第一组：合规样本
•	第二组：抄袭检测样本

用法
•	第一组用于证明“不会乱报”

12. 成功标准

这个 Demo 成功不看模型分数，看演示效果。

成功标准 1

投资人能在 3 分钟内理解产品价值。

成功标准 2

能够明显区分三类结果：
•	正常
•	抄袭
•	围标

成功标准 3

每个结论都有证据支撑，而不是黑盒打分。

成功标准 4

能够说明技术路线未来可扩展到：
•	PDF
•	OCR
•	多模态版面
•	大规模批量审查

13. Demo 演示话术建议

你可以把产品描述为：

这不是传统查重工具，而是一个面向医疗招投标合规审查的智能识别引擎。
它不仅识别文本重复，还能识别洗稿改写、模板同源、团队重叠、报价协同等围标风险，并给出证据链。
