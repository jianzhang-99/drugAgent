# 03 后端实施步骤（可照着做）

## 0. 先复用现有能力

优先复用以下已有类：
- `src/main/java/com/liang/drugagent/config/VectorStoreConfig.java`
- `src/main/java/com/liang/drugagent/service/AgentChatService.java`
- `src/main/java/com/liang/drugagent/prompt/SystemPromptManager.java`

目标：避免重复造轮子，只补场景一缺失模块。

## 1. 建包与骨架

建议新增包：
- `com.liang.drugagent.bidreview.controller`
- `com.liang.drugagent.bidreview.service`
- `com.liang.drugagent.bidreview.domain`
- `com.liang.drugagent.bidreview.repository`

建议先建这些类（空实现也可以）：
- `BidReviewController`
- `BidReviewTaskService`
- `BidDocumentParseService`
- `BidChunkService`
- `BidRetrievalService`
- `BidJudgeService`
- `BidReportService`

完成标准：项目可编译。

## 2. 先做异步任务最小闭环

### Step 2.1 定义任务对象

建议字段：
- `taskId`
- `status`（`PENDING/RUNNING/SUCCESS/FAILED`）
- `progress`（0-100）
- `statusText`
- `errorMessage`
- `createdAt`/`updatedAt`

### Step 2.2 实现内存任务仓库（MVP）

- 用 `ConcurrentHashMap<String, BidReviewTask>` 保存任务状态
- 先不落库，保证开发速度

### Step 2.3 接口先返回假数据

先实现：
- `POST /api/bid/compare`：返回 `taskId`
- `GET /api/bid/task/{taskId}/progress`：返回动态进度

完成标准：前端能稳定轮询。

## 3. 接入文本处理链路

### Step 3.1 文本解析（先易后难）

MVP 阶段：
- 先支持 `.txt/.md`
- 若上传 pdf/docx，先返回“暂不支持”或降级说明

增强阶段：
- 再引入 PDF/DOCX 解析器

### Step 3.2 语义切片

建议规则：
- 目标长度：500-800 token
- 保留 metadata：章节、段落序号、原文偏移
- 切片结果输出 `List<BidChunk>`

完成标准：能打印切片统计日志（总字数、切片数、平均长度）。

## 4. 接入检索与重排

### Step 4.1 基准文档入向量库

- 对基准标书切片做 embedding
- 写入 `VectorStore`
- 任务结束前可调用 `save(File)` 持久化

### Step 4.2 待查文档检索

- 对待查切片逐条召回 TopK（建议 15）
- 记录候选项与初筛分

### Step 4.3 重排与阈值

- 调用重排模型得到精排分
- 过滤阈值建议先用 `0.85`

完成标准：能输出“高危候选对”列表。

## 5. 接入大模型裁决

### Step 5.1 新增场景一 Prompt

在 `SystemPromptManager` 中新增场景一专用常量，要求：
- 明确剔除合法法规引用
- 要求结构化 JSON 输出
- 包含 `risk_level`、`reason` 等字段

### Step 5.2 裁决服务

输入：高危候选对  
输出：结构化裁决对象

建议字段：
- `isPlagiarism`
- `riskLevel`
- `similarityType`
- `reason`
- `suggestion`

完成标准：多组样例下 JSON 解析稳定。

## 6. 报告聚合与结果接口

实现：
- `GET /api/bid/report/{taskId}`

报告至少包含：
- 全局结论（总体风险等级、综合雷同率）
- 分片明细（A片段、B片段、分数、裁决理由）
- 可视化统计（章节分布）

完成标准：前端可直接渲染。

## 7. 每一步自测命令

```bash
# 后端启动
./mvnw spring-boot:run

# 健康检查（示例）
curl http://localhost:8124/api/chat/simple -X POST -H 'Content-Type: application/json' -d '"你好"'
```

## 8. 代码评审必查点

- 异步任务异常是否会写入 `FAILED` 状态
- 空结果是否有可解释返回（不是 500 + 空栈）
- 是否存在硬编码密钥
- 是否有超时和重试策略
