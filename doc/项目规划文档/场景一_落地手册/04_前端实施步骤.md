# 04 前端实施步骤（可照着做）

## 1. 页面目标

新增一个“标书查重工作台”页面，完成 3 个动作：
- 上传并发起查重任务
- 展示任务进度
- 展示报告与双栏高亮

## 2. 路由与页面骨架

当前已有路由：`/drug`、`/compliance`、`/knowledge`。  
新增：`/bid-review`。

建议新增文件：
- `drug-agent-web/src/views/BidReview.vue`
- `drug-agent-web/src/api/bidReview.js`
- `drug-agent-web/src/components/bid-review/UploadPanel.vue`
- `drug-agent-web/src/components/bid-review/ProgressPanel.vue`
- `drug-agent-web/src/components/bid-review/ReportPanel.vue`

## 3. 先做可运行最小版本

### Step 3.1 上传与任务创建

- 表单字段：`fileA`、`fileB`（或 `referenceBidId`）
- 点击开始后调用 `POST /api/bid/compare`
- 保存返回的 `taskId`

### Step 3.2 进度轮询

- 每 2 秒调用一次 `GET /api/bid/task/{taskId}/progress`
- 状态为 `SUCCESS/FAILED` 时停止轮询
- `FAILED` 时展示后端错误信息

### Step 3.3 报告渲染

- 任务成功后调用 `GET /api/bid/report/{taskId}`
- 渲染顶部摘要（风险等级、雷同率、高危段落数）
- 渲染明细列表（支持点击定位）

## 4. 双栏高亮设计建议（MVP）

MVP 不追求字符级定位，先做段落级：
- 左栏：待查片段 A
- 右栏：基准片段 B
- 中间显示分数与裁决标签（`LOW/MEDIUM/HIGH/CRITICAL`）

## 5. 交互细节（建议）

- 上传后立即禁用按钮，避免重复提交
- 任务处理中显示阶段文案（例如：`正在切片...`）
- 结果页支持“复制报告摘要”

## 6. 前端联调检查项

- [ ] 代理配置下 `/api` 能转发到后端
- [ ] 接口异常时有统一提示
- [ ] 页面刷新后能根据 `taskId` 恢复状态（可选）
- [ ] 长文本区域滚动不卡顿
