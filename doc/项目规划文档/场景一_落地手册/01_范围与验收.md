# 01 范围与验收

## 1. 场景目标

在医疗招投标审查中，自动识别标书的雷同风险，重点判断：
- 是否存在模板化复制、参数洗稿、关键指标雷同
- 是否应排除法规条文等合法引用

## 2. 版本边界（必须先做 MVP）

### V1（本次必须完成）
- 输入：两份可解析文本文档（建议先支持 `.md`/`.txt`）
- 核心链路：切片 -> 向量检索 -> 重排 -> 大模型裁决
- 输出：任务进度 + 结构化报告 + 雷同片段列表
- 前端：上传、进度、结果展示

### V2（可选增强）
- 支持复杂 PDF/DOCX 解析与表格还原
- 支持跨多份基准库批量比对
- 支持 SSE/WebSocket 实时推送
- 支持更精细的高亮定位（字符级锚点）

## 3. 非目标（本轮不做）

- 不做生产级权限体系
- 不做多租户隔离
- 不做全量历史报告管理后台
- 不追求一次性支持所有文档格式

## 4. 验收标准（量化）

### 功能验收
- [ ] 成功提交查重任务并返回 `taskId`
- [ ] 能查询任务状态（`PENDING/RUNNING/SUCCESS/FAILED`）
- [ ] 成功返回报告 JSON，字段完整
- [ ] 前端能展示报告并定位关键雷同片段

### 质量验收
- [ ] 单次任务失败不影响其他任务
- [ ] 单任务全链路可在 2-10 分钟内完成（取决于文档长度）
- [ ] 出错时有明确错误信息（不返回空白）
- [ ] 至少有 1 组真实样例可复现结果

## 5. 推荐里程碑门槛

里程碑通过条件：
- M1（基础连通）：Embedding + VectorStore + 基础接口都可调用
- M2（检索链路）：切片、召回、重排可产生候选对
- M3（AI 裁决）：可输出结构化风险结果
- M4（端到端）：前后端完成闭环演示
