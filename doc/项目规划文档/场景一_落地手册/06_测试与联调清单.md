# 06 测试与联调清单

## 1. 测试分层

### 单元测试（后端）
- 文本切片规则测试
- 相似度阈值过滤测试
- 报告聚合逻辑测试

### 集成测试（后端）
- 提交任务 -> 查进度 -> 查报告完整链路
- 模型接口失败时任务状态是否正确落到 `FAILED`

### 前端联调测试
- 上传成功/失败
- 进度轮询停止条件
- 报告字段缺失时页面降级展示

## 2. 场景化测试用例（建议最少 6 条）

1. 正常用例：两份文本有明显雷同，结果应含高风险项
2. 合法引用用例：法规条文重合，不应全部判为高风险
3. 低相似用例：两份文本主题不同，风险应低
4. 异常用例：上传空文件，接口应返回可解释错误
5. 超时用例：模拟模型超时，任务状态应为 `FAILED`
6. 大文档用例：验证进度展示与页面不卡顿

## 3. 联调步骤

1. 后端先用固定样例产出稳定 JSON
2. 前端先按固定 JSON 渲染（不等真接口）
3. 再切换真接口联调
4. 最后做异常路径回归

## 4. 发布前检查

- [ ] `.yml` 中无真实密钥提交
- [ ] 日志中不打印敏感原文
- [ ] 关键接口有超时保护
- [ ] 失败任务可追踪（有 taskId 与错误信息）
- [ ] 至少一次完整演示录屏或截图留档
